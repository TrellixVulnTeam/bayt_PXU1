Edit the Default Organization

The default organization is identified by the service provider who purchased the PW EMS software. Use this task to edit the default organization. Note that this default organization cannot deleted.

  1. Click the Navigation Bar menu and select Settings > RBAC.
  2. In the Managing Organization window, click the Actions column button in the default organization (Parallel Wireless) row.
  3. Select Edit from the drop-down list that appears.
  4. In the Edit Organization's Properties dialog box, complete the following fields:

    Field

    Description
    Rename the Organization Rename the default service provider name (Parallel Wireless) to the service provider that owns the PW EMS.
    Change Organization Type drop-down list

    Not configurable.
    Change Organization's Admin Email

    Enter the appropriate contact email for the MNO or MVNO service provider organization administrator operator.
    Choose Authentication Type drop-down list

    Select from the following authentication options:

    Note: If LDAP is selected, additional LDAP fields are displayed. LDAP fields are not displayed if Embedded and Managed Authorization is selected.

    Local- Users who login to the PW EMS locally on this PW EMS server. Passwords are managed locally.

    LDAP- Active directory users are authenticated by a Lightweight Directory Access Protocol (LDAP) server and their membership of groups (roles) are examined when authentication is successful. The Active Directory returns a list of groups, to which the authorized user belongs. The externally authorized user is also added to a local database, and is removed when this user logs out, or its session terminates. All user login activity is logged. The LDAP server manages authorization, password rules, security, and so on.
    LDAP Server IP field Enter the IP address for the LDAP server. This provides the PW EMS a way to connect to the LDAP server.
    LDAP Server Port field The default LDAP port 389 appears in this field by default.
    LDAP Server Search Domain field The string used to specify the base domain name (DN), Common Name (CN), unique identifier (UID), and LDAP attributes used for search. A valid DN can be a Domain Component (DC) which uses DNS to define its namespace. The following is an example of a search string: CN=Users,DC=lab,DC=parallelwireless,DC=de
    LDAP Protocol field Not configurable. TCP is the supported protocol on which LDAP runs at this time.
    LDAP User Name field Enter an LDAP user name known to the administrator.
    LDAP User Password field Enter an LDAP password known to the administrator for the user.
  5. If you selected LDAP for your authentication method, click TEST LDAP to verify that the LDAP IP, Port, Protocol, and Search Domain are correct.
  6. Click UPDATE. The organization displays in the Organizations tab.
