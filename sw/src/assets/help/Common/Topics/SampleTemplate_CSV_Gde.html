SampleTemplate CSV File Guide

The Parallel Wireless comma-separated values (CSV) file template (SampleTemplate.csv) is used to create a custom file that contains vBBU, vNode and cell device data, which is saved in tabular format.

Basic CSV Information

The CSV File template is offered to ensure that the customized CSV file template that you create is free from errors and validates properly when it is uploaded to the PW EMS and used to bulk configure the intended devices. The following rules are followed in order to create a valid CSV file:

  • The CSV file can contain up to 100 unique devices. A CSV file row entry can be used for a VBBU, vNode, or cell, so the total number of entries in the file can be more than 100.
  • The CSV file cannot be empty.
  • The PW EMS validates all CSV file rows for the proper headers, mandatory fields, and system profile information.
  • If any user uploads a file and starts the creation process, other users can see status for this file.

CSV File Criteria

  • Device Criteria - Each row in the CSV file is for a vBBU or vNode. For each vNode entry, service for 2 LTE cells and 1 GSM cell is allowed.
  • Multiple-cell Criteria - If a value in the DeviceName column is same for multiple rows then all rows become cells of a Vnode.

CSV Formatting Considerations

  • A colon (:) is used for separating a key value (for example: key:value)
  • A semi-colon (;) is used for separating different profile types (for example: heartbeat:HeartBeat;statsinterval:StatsInterval).
  • System Profile Format - <ProfileType1>:<ProfileValue1>;<ProfileType2>:<ProfileValue2>; (for example: GsmConfig:bcgsmconfig;HeartBeat:bcheartbeat;).
  • Service Profile Format - <ProfileValue1>;<ProfileValue2>; (for example: LTE Default;LTE Cell1)

Mandatory Columns

The following columns must be completed in the customized CSV file:
Column Description
DeviceName The unique, user-specified name of the device. This column identifies whether a vNode has a single cell or multiple cells. If a vNode has two rows that have the same value in this column, the single device is created with two cells. If both rows have a unique value, two vNodes are created.
HngName Applies to COM_VNODE only for the HetNet Gateway. Default is the only valid value at this time.
Region Applies to COM_VBBU only for the geographical region in which this vBBU site exists. This column is not used for vNode devices.
Services.CellName Applies to COM_VNODE only. The service is unique to the vNode device and can be either GSM (2G) or LTE (4G).
Services.Profiles Applies to COM_VNODE only. Accepted service profiles are:
  • GSM Default
  • LTE Default
  • LTE Cell1
  • LTE Cell2
  • GSM GPRS Custom
  • LTE Cell Band8
  • LTE 41733-CI9999-865621053
  • LTE 41733-CI9999-865931053
  • LTE 41733-CI9999-417069499
  • GSM 41733-CI9999-417069500
Site Applies to COM_VBBU only for a specific site location. This column is not used for vNode devices.
VbbuDeviceId Applies to COM_VNODE only for the hardware (HW) Class ID that was configured in the WCT vBBU interface. For example, the Parallel Wireless HW Class ID is C19999.
DeviceType Accepted device types are:
  • COM_VBBU
  • COM_VNODE
ProductClass

The device ID for the vBBU or vNode. This is the HW Class ID that was configured in the WCT vBBU interface for a vNode that was selected there. For example, the Parallel Wireless HW Class ID is C19999.
SerialNumber The device serial number, which is unique to the device.
Services.DataModelName

Applies to COM_VNODE only. This (LTE or GSM) value must be correct according to the existing PW EMS database service definition profiles. To look up these values, go to Devices > Service Definitions.

Services.Version Applies to COM_VNODE for software version of the cells on the vNode.
SystemInformationCreate.Version Applies to COM_VNODE only for the software version of the vNode.

Columns that Require Numbers Only

Column Entry Description
DownlinkBandwidth The downlink bandwidth value from 1 to 2000 minutes for the vBBU or vNode.
Services.AdditionalConfigurations.CellConfig.GSM.GPRS.BVCI Defines the Base Station System General Packet Radio Service (GPRS) Protocol (BSSGP) Virtual Connection Identifier from 2 to 65535 toward the SGSN.
Services.AdditionalConfigurations.CellConfig.GSM.GPRS.NSVCI Defines the Network Service Virtual Connection Identifier (NSVCI) from 2 to 65535 toward the SGSN.
Services.AdditionalConfigurations.RRHCarrierIdx Defines the LTE 4G Radio Access Network (RAN) Radio Frequency (RF) Remote Radio Head (RRH) Carrier Index number.
Services.AdditionalConfigurations.CellConfig.LTE.RAN.Common.CellIdentity Defines the cell identity LTE 4G RAN common cell identity value, which has value of null, which is no value.
Services.AdditionalConfigurations.CellConfig.LTE.RAN.RF.EARFCNDL

The Absolute Radio Frequency Channel Number (EARFCN) required for the downlink, which is used to calculate the center frequency for the EUTRA from 0 to 65535. This required value is based on the country of operation and hardware class ID of the node.

Columns that Require True or False (Boolean) Values

Column Entry Description
SystemInformationCreate.IsPKIEnabled Enter true or false to indicate if PKI is enabled on the device.
AdditionalConfigurations.MultiCellNeighbors

Enter true or false to indicate that a neighboring cell has multiple cells.
Services.AdditionalConfigurations.BidirectionalANR Enter true or false to indicate if bi-directional Automatic Neighbor Relations (ANR) is enabled with this cell.
Services.AdditionalConfigurations.ManualANR Enter true or false to indicate if ANR operations are enabled to be performed manually.
Services.AdditionalConfigurations.X2ANR Enter true or false to indicate if the ANR of X2-reported neighbors is enabled so that other cells can contact a cell to report themselves as neighbors.
Services.AdditionalConfigurations.UEReportedANR Enter true or false to indicate if User Equipment (UE) ANR reporting is enabled for this cell.
