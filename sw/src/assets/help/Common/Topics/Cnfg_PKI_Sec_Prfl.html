Configure a PKI Security Profile

Use this task to create a system security profile which includes the PKI certificate parameters required for server authentication.

  1. In the navigation bar, click Profiles > Summary.
  2. In the Profiles window, click the System Profiles tab.
  3. Click +Create.
  4. In the Create System Profile window, enter the name of this profile in the Profile Name field.
  5. Select PkiSecurity from the Profile Type and complete the following fields:
    NOTE: Use the Name field to search for a specific parameter on this page.

    Field

    Description
    Security Profile Name Enter the PKI security profle name with a maximum of 128 characters.
    Factory Domain (Optional) Enter restrict.parallelwireless.com as the domain name for access devices to obtain IPSec Factory certificates in conjunction with the PKI infrastructure.
    OperDomain (Optional) Enter the Fully Qualified Domain Name (FQDN) as the operational domain in 4 to 64 characters [host name.[domain].[tid] for access devices obtain an Operational certificate in conjunction with the PKI infrastructure.
    OperCACert (Optional) Paste the CA certificate file from a local system directory. A maximum of 4096 characters is permitted for this string.
    RenewalThreshold (Optional) Specify the number of seconds from 50 to 99 for the X.509 digital certificate renewal threshold. The default threshold is 80 seconds.
    Validity Timer (Optional) Specify the End Entity (EE) certificate validity time from 10 to 3600 minutes. The default EE certificate validity value is 60 minutes.
    KurEnabled checkbox (Optional) The Key Update Request (KUR) checkbox is disabled by default. Check the checkbox to allow the end entity to request a key update when a key pair is due to expire automatically.
    SrvlpAddr (Optional) Specify a valid CMPv2 server IP address.
    srvPort Number (Optional) Specify the server HTTP (80) or HTTPS (443) port from 0 to 65535. The default port is 80.
    SrvDirPath (Optional) Specify the directory path of the server from 3 to 200 char-acters. If configured, this path is considered in the formation of the URL.
    SrvUrl (Optional) Specify the CMPv2 server URL.
    AuthType Enter the 128 character RSA public-key algorithm authentication for the Certificate Management Protocol (CMPv2) server.
    UserName (Optional) Specify the valid CMPv2 server-supplied user name using a unique alpha-numeric string for server authentication.
    Password (Optional) Specify the valid CMPv2 server authentication password using a unique alphanumeric string.
    RevokePassword (Optional) Specify the valid CMPv2 server-supplied revocation password (a unique alphanumeric string).
    CaCertHsh (Optional) Specify the type of CA certificate fingerprint hash algorithm type. That is: MD5 message-digest algorithm (md5 which is the default), SHA-1 (Secure Hash Algorithm 1) (sha1), or SHA-2 (Secure Hash Algorithm 2) (sha2).
    CaCertUrl Enter the URL of the CA certificate, which is an IPv4 address in the format http://<url>
    CaCertType (Optional) Specify the type of CA certificate fingerprint hash algorithm type. That is: MD5 message-digest algorithm (md5 which is the default), SHA-1 (Secure Hash Algorithm 1) (sha1), or SHA-2 (Secure Hash Algorithm 2) (sha2).
    Years Specify the number of years from 0 to 20 for which the EE certificate is valid.
    Days Specify the number of days from 0 to 365 for which the EE certificate is valid.
    Hours Specify the number of hours from 0 to 23 for which the EE certificate is valid.
    Minutes Specify the number of minutes from 0 to 59 for which the EE certificate is valid.
    PkcAlgo The default RSA (Rivest–Shamir–Adleman) public-key algorithm that is used for secure data transmission size is a numeric value of 101. A value between 101 and 106 can be entered if you want to change the default value.
    PKcKeySize The default RSA (Rivest–Shamir–Adleman) public-key size that is used for secure data transmission is 1024. A value of 2048 or 4096 can be entered if you need to change the default value.
    CountryName (Optional) Specify a unique alphanumeric EE Distinguished Name (DN) for the country (C) designation. For example, USA
    StateName (Optional) Specify a unique alphanumeric EE DN for the province or state (ST). For example: MA
    LocalityName (Optional) Specify a unique alphanumeric EE DN for the locality (L) designation. For example, Boston
    OrgName (Optional) Specify a unique alphanumeric EE DN for the organization (O) designation. For example: Company.
    OrgUnitName (Optional) Specify a unique alphanumeric EE DN for the organizational unit (OU) designation. For example: Security
    SubAltNameIp (Optional) Enter a subject alternative name host IP address to be pro-tected by this certificate.
    SubAltNameDns1 (Optional) Specify the Subject Alternative Name (SAN) (DNS1) FQDN 1 that allows an additional host name to be protected by this certificate. The SAN is part of X.509v3 extensions in a digital certificate and is used during authentication of the HetNet Gateway (HNG). An EE Subject can have multiple alternate names; therefore, there is a need to have unique identification for IPsec tunnels connecting to an IPsec server.
    SubAltNameDns2 (Optional) Enter the SAN 2 (DNS2) FQDN 2 allows a second additional host name to be protected by this certificate.
  6. Click Create+.
