Manage Alarms

  1. In the navigation bar, click Alarms.
  2. In the Alarms window, you can choose how to display alarms and take certain actions with alarms by using the alarm configuration bar selections, which are shown below and discussed in more detail in the following steps.
  3. In the Alarms window, the following table describes the default and optional columns that appear in the All Alarms table.
    NOTE: You can control how default or optional columns appear in the All Alarms table by clicking the column display icon . In the drop-down list, check or uncheck the checkbox for any the columns described below that you want to filter in the table. Unchecking a checkbox hides a column and checking a checkbox makes a column appear in the table.

    Field

    Description
    Checkbox

    If you want to download alarms, check the checkbox next to one or more alarm entry and click the download icon. In the pop-up menu that displays, select either Excel for an Alarms.xlsx file or CSV for an Alarm.csv file. The file you select downloads to your web browser's default download location on your system.
    Alarm Name (Default column) Displays a description of the alarm. For example, OLD_CERT_FIND_FAILURE
    Severity

    (Default column) Displays the severity of the alarm. Click the Severity drop-down list and check the check box next to each of the following alarm severity levels to filter alarms columns that you want to appear in the table:

    • Critical - Action must be taken immediately. If no actions are taken, there may be physical, permanent, and irreparable damage to your system.
    • Major - Critical conditions exist. The functionality has been seriously compromised and a loss of functionality, hanging applications, and dropped packets may occur. If no actions are taken, your system suffers no physical harm, but ceases to function.
    • Minor - Error conditions exist. The functionality has been impaired to a certain degree and you might experience compromised functionality. There is no physical harm to your system, but you need to take actions to keep your system operating properly.
    • Warning - Warning conditions exist. Some irregularities in performance. These conditions are noteworthy and you should take actions to keep your system operating properly.

    Any alarm of a checked alarm severity level appears in the table. Uncheck the checkbox of an alarm severity level to hide any alarms that belong to this alarm type so they do not appear in the table.
    Description (Default column) Describes in plain language what the alarm means. For example, "Unable to find old certificates."
    Message (Default column) Displays the system message ID and type of error in detail.
    Time Stamp (Default column) Displays the day, month, year, and Greenwich Mean Time (GMT) time for when the alarm was triggered.
    Device Name (Optional column) Displays the device name that triggered the alarm. This name may display the same as the Device ID, if it was not re-named by a user.
  4. Click an alarm once to provide a detailed alarm view. For example, a critical alarm is expanded to show detailed information for it:
    The fields described in the following table appear only in the detailed alarm view:

    Field

    Description
    Object ID The internal path for the object involved.
    Raised by Indicates the entity that raised the alarm.
    Device ID This is a combination of the Organization Unique ID (OUI) for Parallel Wireless (41733), product class (C19999) and product serial number (for example, S3837C2X1113972) that were made.

    For example the complete Device ID displays like this: 41733-C19999-S3837C2X1113972

    Functional Area Displays what type of service or network element is involved.
    Device Name Displays the device name that triggered the alarm. This name may display the same as the Device ID, if it was not re-named by a user.
    Alarm ID Unique alarm identifier (ID) generated and assigned by the PW EMS.
    Acknowledge Indicates whether the alarm has been acknowledged or not. Click the Acknowledge drop-down list and check the checkbox for the ACK (acknowledged) and UNACK (unacknowledged) options to display them in the table. Uncheck the checkbox to hide either of these alarm types so they do not appear in the table.
    Source
    Identifies the source device type of the alarm. Click the Source drop-down list and check the checkbox for the following source device types you want to filter from which the PW EMS receives alarms:
    • ACS
    • COM_VBBU
    • COM_VNODE
    • COM_VNODE_2G
    • COM_VNODE_4G
    • HNG

    Any checked device appears in the table. Uncheck the checkbox to hide source device alarm types so they do not appear in the table.
  5. Check the checkbox of one or more alarms to perform actions on them.
  6. Click the Action drop-down list and select one of the following actions to perform on the alarm(s) that you selected:

    Alarm Action

    Description
    ACK Check the checkbox to acknowledge the alarm(s) you selected in the previous step.
    UNACK Check the checkbox to unacknowledge the alarm(s) you selected in the previous step.
    CLEAR Check the checkbox to clear the alarm(s) you selected in the previous step.
  7. Click More to select from the following alarm filter types that you want to use to filter or winnow the type of alarm information that you want to see in the All Alarms table. The alarm filter types shown in the following figure are described in the following table:

    Alarm Filter

    Description
    Alarm Name Enter a full or partial alarm name from which you want to filter alarms.
    Device Name Enter a device name from which to filter alarms.
    Device ID Enter the device ID from which to filter alarms.
    Alarm ID Enter the alarm ID from which to filter alarms.
    Message Enter message text from which to filter alarms.
    Description Enter a full or partial description from which to filter alarms.
    Timestamp Select a time (from and to) from the date calender and enter the time in hours, minutes and seconds (HH:MM:SS) and click Apply from which to filter alarms.
  8. To sort table entries, position the cursor to the right of any column header until a gray arrow appears and click this arrow up (beginning of entry list) or down (end of entry list) to display results or vice versa, as shown below:
  9. Click the Auto Refresh slider On if you want to have the Alarms window refresh periodically once every 10 minutes. Click Off to return to the default for this page setting.
  10. Click LOGS at the top of the window to use as a short-cut to the Audit Logs page. This page can also be accessed from the navigation bar > Audit Logs.
