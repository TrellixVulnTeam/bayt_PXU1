Define User Roles for an Organization

Role-based Access Control (RBAC) is a way to restrict system access to authorized users by determining user roles and the privileges they are allowed to perform on the PW EMS. The user roles described in the following sections are typical in service provider networks. You can specify the following user roles for your network or customize user roles to fit the characteristics of your network.

Network System Engineer

This super user performs the following tasks:

  • Installs and configures intelligent dashboards and O-RAN elements on the PW EMS system, which includes vBBU access nodes, the HetNet Gateway (HNG), Evolved Packet Core, and Integrated Management System (IMS) systems.
  • Operates, manages, and scales current service provider networks to add additional capacity, coverage, and enable different functions.

Network Architect

This user performs the following tasks:
  • Designs, plans and configures the network so that it can scale properly by supporting new user demands across regions, and helps to operate the network.
  • Troubleshoots any network problems that occur by producing network-level statistics, KPIs, and heat maps to help service providers understand the current operational constraints.

Network Operator

This service provider user performs the following tasks:
  • Manages configurations and performs maintenance on the PW EMS.
  • Uses dashboards to monitor system changes, network performance, and troubleshoots alarms and events daily. Dashboards help isolate areas of the network that need attention and determine remedial actions to resolve problems and affect changes across a large number of network elements.

IT Engineer

This IT user performs the following network infrastructure tasks, which may overlap with those of a security engineer or system administrator:
  • Configures RBAC for all user types, which includes the configuration of private and public cloud native technologies for service provider network solutions.
  • Connects the PW EMS to back-end databases, authentication servers, logging servers and other systems.
  • Reviews system capacity requirements.
  • Configures PW EMS system parameters.
  • Views alarms and monitors all connectivity related to IT systems.

Security Engineer

This security administrator user performs the following tasks, which may overlap with those of an IT engineer or system administrator:
  • Configures and manages security policies, certificates, RBAC for all user types, user names and their passwords, and key management to secure all network elements.
  • Coordinates with the IT Engineer to setup access and interoperability with external Public Key Infrastructure (PKI), Authentication, Authorization, and Accounting (AAA) servers, and logging servers.
  • Uses dashboards to monitor security breaches, expiring certificates that need to be renewed, and so on.

Technical Operations Manager

This user performs the following tasks:
  • Focuses on alerts and events that need immediate action and helps to deliver and enforce service-level agreement (SLA) commitments between service providers and their clients. The key stakeholder may be business manager and other executive leadership teams.
  • Accesses high-level custom dashboards to gain actionable intelligence necessary to manage the network with collaborative KPIs, alarms to measure system health such as downtime, bandwidth enforcement, and end user connectivity.

Business Manager

This user performs the following tasks:
  • Manages consumer network demand and be able to plan for the expansion of the network.
  • Delivers revenue-generating services for a business and their customers.
  • Reviews and acts on service and revenue issues that may affect SLAs by monitoring KPIs and dashboards.
  • Anticipates network failures.
  • Customer Support Manager
  • This user resolves customer reported issues by tracing calls, analyzing event logs, and reviewing alarms.

Customer Support Manager

This user resolves customer reported issues by tracing calls, analyzing event logs, and reviewing alarms.

Network Engineer

This technical user performs the following tasks which may overlap with those of a network operator:
  • Uses their high-level access to the PW EMS system to resolve any end-to-end network issues by using packet tracing tools, determining network device reachability, and reviewing network statistics that are associated with network protocols.
  • Navigates to specific interfaces and ports on the network to determine the status of certain networking protocols for determining network connectivity.
  • Uses dashboards to monitor system changes, network performance, and troubleshoot alarms and events daily. Dashboards help isolate areas of the network that need attention and determine remedial actions to resolve problems and affect changes across a large number of network elements.
